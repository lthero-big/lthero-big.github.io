Guidance: a cheat code for diffusion models

"Classifier-free guidance"显著改善了"conditional diffusion models"生成的图像（几乎没有额外成本）。它实施简单且极其有效。它也是OpenAI的DALL·E 2和Google的Imagen的关键组成部分，为它们的惊人图像生成结果提供动力。在这篇博文中，我将分享我的观点，并尝试解释它的工作原理。

Diffusion guidance

仅仅两年前，Diffusion guidance还是生成建模研究的边缘领域，但如今，Diffusion Model已经成为图像和音频生成的首选模型。

它们对高维数据分布p(x)进行建模。与尝试直接逼近p(x)（这是基于似然的模型所做的）不同，它们试图预测所谓的得分函数，∇xlogp(x)∇_x log p(x)∇x​logp(x)

要从Diffusion Model中取样，首先将"input images"初始化为随机噪声，然后通过朝着"得分函数（score function）"的方向（对数似然增加最快的方向）进行迭代去噪，同时加入一些额外的随机噪声以避免陷入分布的模态中。这被称为随机梯度 Langevin 动力学（SGLD）

在Conditional Diffusion Models中，我们有一个额外的输入y（例如， a class label 或 a text sequence），我们试图建模条件分布p(x∣y)p(x|y)p(x∣y)。实际上，这意味着要学会预测 条件得分函数∇xlogp(x∣y)∇_xlog p(x|y)∇x​logp(x∣y)。

得分函数的一个巧妙之处在于它对于分布的归一化是不变的：如果我们只知道分布p(x)与一个常数因子的关系，即p(x)=p^(x)Zp(x)=\hat p(x)Zp(x)=p^​(x)Z，并且我们只知道p^(x)\hat p(x)p^​(x)，那么我们仍然可以计算得分函数：

∇xlogp^(x)=∇xlog(p(x)⋅Z)=∇x(logp(x)+logZ)=∇xlogp(x)∇_xlog \hat p(x) = ∇_xlog (p(x)⋅Z) = ∇_x(log p(x) + log Z) = ∇_xlog p(x)∇x​logp^​(x)=∇x​log(p(x)⋅Z)=∇x​(logp(x)+logZ)=∇x​logp(x),

在这里，我们利用了梯度算子的线性性质，以及规范化常数Z=∫p^(x)dxZ=∫\hat p(x)dxZ=∫p^​(x)dx不依赖于x（因此其对x的导数为零）的事实。

非规范化概率分布经常出现，因此这是一个有用的特性。对于条件模型，它使我们能够应用贝叶斯定理将得分函数分解为无条件分量和混入条件信息的分量：

p(x∣y)=p(y∣x)⋅p(x)p(y)⟹logp(x∣y)=logp(y∣x)+logp(x)−logp(y)⟹∇xlogp(x∣y)=∇xlogp(y∣x)+∇xlogp(x)p(x|y) = p(y|x)⋅p(x)p(y) ⟹ log p(x|y) = log p(y|x) + log p(x) - log p(y) ⟹ ∇_xlog p(x|y) = ∇_xlog p(y|x) + ∇_xlog p(x)p(x∣y)=p(y∣x)⋅p(x)p(y)⟹logp(x∣y)=logp(y∣x)+logp(x)−logp(y)⟹∇x​logp(x∣y)=∇x​logp(y∣x)+∇x​logp(x),

在这里，我们使用了∇xlogp(y)=0∇_xlog p(y) = 0∇x​logp(y)=0。换句话说，我们可以将条件得分函数简单地视为无条件得分函数和一个条件项的总和。（请注意，条件项∇xlogp(y∣x)∇_xlog p(y|x)∇x​logp(y∣x)本身不是得分函数，因为梯度是对x而不是y的。）

在整篇博文中，我大多数时候都忽略了扩散模型估计的分布的时间依赖性。这使我不必在每个步骤上添加额外的条件变量和下标。在实践中，扩散模型执行迭代去噪，因此通常在每个步骤上都受到输入噪声水平的条件限制。

Classifier guidance

分类器引导（Classifier Guidance）提供了一种引导Diffusion的去噪过程（reverse process)朝着最大化最终样本被分类为特定类别的概率的方向进行的方法(默认随便生成内容，现在要指定内容)。这可以使样本反映出“在训练期间未提供给扩散模型的任何类型的条件信号”。换句话说，它使得在训练之后附加条件成为可能。

对于分类器引导，我们需要一个辅助模型来预测 p(y∣x)p(y|x)p(y∣x)，其中 y 表示任意输入特征，可以是类别标签、文本描述（a class label 或 a text sequence），甚至是更结构化的对象，如分割图或深度图。我们将这个辅助模型称为分类器，但请记住，我们可以使用许多不同类型的模型来实现这一目的，不仅仅是狭义上的分类器。这样的设置的一个好处是，这些辅助模型通常比Diffusion Model更小且更容易训练。

文字解释

首先要注意的是，p(y∣x)p(y|x)p(y∣x)正是分类器和其他判别模型试图拟合的内容：x是某个高维输入，y是目标标签。如果我们有一个可微的判别模型来估计p(y∣x)p(y|x)p(y∣x)，那么我们也可以轻松地获得∇xlogp(y∣x)∇_xlog p(y|x)∇x​logp(y∣x)。

要将一个无条件扩散模型转变为有条件模型，我们只需要一个分类器！

扩散模型可以事后通过这种方式进行条件化的观察是由Sohl-Dickstein等人4和Song等人5提到的，但Dhariwal和Nichol6真正强调了这一点，并展示了"Classifier guidance"如何通过增强条件信号而显著提高样本质量，即使与传统的条件建模结合使用时也是如此。为了实现这一点，他们通过一个因子来缩放条件项：

∇xlogpγ(x∣y)=∇xlogp(x)+γ∇xlogp(y∣x)∇_xlog pγ(x|y) = ∇_xlog p(x) + γ∇_xlog p(y|x)∇x​logpγ(x∣y)=∇x​logp(x)+γ∇x​logp(y∣x),

γγγ称为引导尺度（guidance scale），若γ>1会放大条件信号的影响。它非常有效，尤其是与GANs7的截断技巧相比，后者有类似的目的。

如果我们恢复从贝叶斯定理到分类器引导的梯度和对数运算，就更容易理解发生了什么：

pγ(x∣y)∝p(x)⋅p(y∣x)γpγ(x|y)∝p(x)⋅p(y|x)γpγ(x∣y)∝p(x)⋅p(y∣x)γ

优点

在语言建模中，现在已经普遍采用一种方法，即首先训练一个强大的无条件语言模型，然后根据需要对其进行下游任务的自适应（通过少量样本学习或微调）。表面上看，"Classifier guidance"似乎为图像生成实现了类似的功能：我们可以训练一个强大的无条件模型，然后在测试时使用一个单独的分类器进行条件化。

缺点

一个重要的注意点是，我们将对带有不同噪声水平的嘈杂输入 xtx_txt​ 应用这个辅助模型，因此它必须对这种特定类型的输入失真有鲁棒性。这似乎排除了使用现成的分类器，并意味着我们需要训练一个定制的抗噪声分类器，或者至少对现成的分类器进行微调以使其具有抗噪声能力。我们还可以明确地使分类器条件化于时间步 t，这样就不必仅从输入 xtx_txt​ 中推断噪声水平。

但即使我们有一个抗噪声的分类器，分类器引导在其有效性方面存在固有的限制：输入x中的大多数信息与预测y无关，因此，关于分类器对其输入的梯度可以在输入空间中产生任意（甚至是对抗性的）方向。

事实证明，我们可以通过将现成的分类器（期望无噪声输入）与我们的Diffusion Model相结合来构建一个合理的抗噪声分类器。而不是将分类器应用于 xtx_txt​ ，我们首先使用Diffusion Model预测 x^0\hat x_0x^0​，并将其作为分类器的输入。 x^0\hat x_0x^0​仍然是扭曲的，但是通过模糊而非高斯噪声。现成的分类器通常对这种失真更有鲁棒性。Bansal 等人将这个技巧称为“前向通用引导”，尽管这已经为人所知。他们还建议了一些更高级的后期引导方法。

使用分类器，我们现在可以确定输入空间中最大化条件信号对数似然的方向，只需计算关于 xtx_txt​ 的梯度：∇xtlogp(y∣xt)∇_{x_t} log p(y|x_t)∇xt​​logp(y∣xt​)。（注意：如果我们使用上述技巧从现成的分类器构建一个抗噪声分类器，这意味着我们需要通过扩散模型进行反向传播。）

因此，采样过程将通过输入空间追踪不同的轨迹。为了控制条件信号对采样过程的影响，我们可以通过一个因子 γ 来缩放分类器梯度的贡献，这被称为引导比例：

组合更新方向将更强烈地受到来自分类器的方向的影响（提供 γ > 1，这通常是这样的）：

这个缩放因子 γ 是一个重要的采样超参数：如果它太低，效果将不明显。如果它太高，样本将会变形且质量低下。这是因为从分类器获得的梯度并不一定指向位于图像流形上的方向——如果我们不小心，可能实际上会进入对抗性示例，这些示例最大化了类别标签的概率，但实际上并不像该类别的任何示例。

在我之前关于扩散引导的博客文章中，我阐述了输入空间中的这些向量操作与它们对应的分布操纵之间的联系。简要回顾一下这种联系以使其更加明显：

我们采取了从扩散模型获得的更新方向，对应于 ∇xtlogpt(xt)∇_{x_t} log pt(xt)∇xt​​logpt(xt) （即分数函数），以及从分类器获得的（缩放的）更新方向， γ⋅∇xtlogp(y∣xt)γ ⋅ ∇_{x_t} log p(y | x_t)γ⋅∇xt​​logp(y∣xt​) ，并通过简单地将它们相加来组合它们： ∇xtlogpt(xt)+γ⋅∇xtlogp(y∣xt)∇_{x_t} log pt(x_t) + γ ⋅ ∇_{x_t} log p(y | x_t)∇xt​​logpt(xt​)+γ⋅∇xt​​logp(y∣xt​)。

这个表达式对应于 $pt(xt) ⋅ p(y | xt)γ $的对数的梯度。

换句话说，我们有效地重新加权了模型分布，根据分类器为所需类别标签分配的概率改变了每个输入的概率。

引导缩放 γ 对应于分类器分布的温度。高温度意味着相对于其他输入，分类器分配高概率的输入被更激进地加权。

结果是一个更有可能产生与所需类别标签一致的样本的新模型。

Classifier-free guidance

正如名称所示，它不需要训练一个单独的分类器。它只需要对代码进行一行更改。在训练期间(随机drop out the conditioning)和在采样期间(mix the conditional and unconditional score estimates)，

使用"conditioning dropout"性质的条件扩散模型p(x|y)进行训练，前提是：在某些时间段内，将条件信息y被删除（通常效果良好的是10-20%）。

文字解释

在实践中，条件信息y通常被替换为代表没有条件信息的特殊输入值（Null值）。由此产生的模型现在可以同时作为条件模型p(x|y)和无条件模型p(x)运行，具体取决于是否提供了条件信号。也许你会认为这会以牺牲条件建模性能为代价，但实际上，实际中的影响似乎微不足道。

这给我们带来了什么好处呢？回想一下之前的贝叶斯定理，但让我们以另一种方式应用它：

p(y∣x)=p(x∣y)⋅p(y)p(x)⟹logp(y∣x)=logp(x∣y)+logp(y)−logp(x)⟹∇xlogp(y∣x)=∇xlogp(x∣y)−∇xlogp(x)p(y|x) = p(x|y)⋅p(y)p(x) ⟹ log p(y|x) = log p(x|y) + log p(y) - log p(x) ⟹ ∇xlog p(y|x) = ∇xlog p(x|y) - ∇xlog p(x)p(y∣x)=p(x∣y)⋅p(y)p(x)⟹logp(y∣x)=logp(x∣y)+logp(y)−logp(x)⟹∇xlogp(y∣x)=∇xlogp(x∣y)−∇xlogp(x).

我们已经将条件项表示为条件和无条件得分函数的函数，而我们的扩散模型提供了这两者。现在我们可以将其代入分类器引导的公式中：

∇xlogpγ(x∣y)=∇xlogp(x)+γ(∇xlogp(x∣y)−∇xlogp(x))∇xlog pγ(x|y) = ∇xlog p(x) + γ(∇xlog p(x|y) - ∇xlog p(x))∇xlogpγ(x∣y)=∇xlogp(x)+γ(∇xlogp(x∣y)−∇xlogp(x))，

或者等价地：

∇xlogpγ(x∣y)=(1−γ)∇xlogp(x)+γ∇xlogp(x∣y)∇xlog pγ(x|y) = (1-γ)∇xlog p(x) + γ∇xlog p(x|y)∇xlogpγ(x∣y)=(1−γ)∇xlogp(x)+γ∇xlogp(x∣y)

这是条件和无条件得分函数的重组。其中， γ是引导尺度（guidance scale），如果γ=0，我们得到无条件模型；如果γ=1，我们得到标准条件模型。

但γ>1才是神奇的地方，当γ>1时，无条件得分变成负数，它降低了样本的无条件可能性，同时增加了条件可能性，使得样本向着有条件的方向生成。

Classifier-free guidance通过降低具有负得分项的无条件可能性来实现这一点，据我们所知，这还没有被探索过，可能会在其他应用中找到用途。

原文：Classifier-free guidance accomplishes this by decreasing the unconditional likelihood with a negative score term, which to our knowledge has not yet been explored and may find uses in other applications.

"Classifier-free guidance"往往比"Classifier guidance"工作得更好，因为贝叶斯分类器比单独训练的分类器更加稳健，且结果更新方向不太可能是对抗性的。另外，它不需要辅助模型，且生成模型可以通过训练期间的条件化dropout简单地与"Classifier-free guidance"兼容。另一方面，这意味着我们不能用它进行事后条件化——所有条件信号都必须在生成模型本身的训练期间可用。

缺点

Classifier-free guidance的一个缺点是采样速度。一般来说，分类器可以比生成模型更小更快，所以Classifier guidance的采样可能比Classifier-free guidance的更快，因为后者需要运行扩散模型的两个前向传递，一个是条件分数，另一个是无条件分数。通过改变体系结构，在网络的后期注入调节，可能会减轻对扩散模型进行多次传递的必要性

Why Classifier-free guidence

为什么这种方法比分类器引导效果好得多呢？主要原因是我们是从一个生成模型构建了这个“分类器”。标准的分类器可以采取捷径（shortcuts），并忽略大部分输入x，同时仍然能够获得竞争性的分类结果不同，生成模型无法拥有这种便捷（shortcuts）。这使得生成的梯度更加稳健(robust)。作为额外的好处，我们只需要训练一个（生成）模型，并且"conditioning dropout"非常容易实现。

值得注意的是，在发布"Classifier-free guidence"思想和OpenAI的GLIDE模型之间只有非常短暂的过度期，GLIDE模型充分利用了这一思想，以至于有时人们将这一思想归功于OpenAI！简单而强大的思想往往会迅速被采纳。在功效与简单性的比率方面，依我看，"Classifier-free guidence"与"conditioning dropout"一样，是真正的改变者！

（实际上，GLIDE的论文中提到，他们最初训练了一个文本条件模型，并且只在微调阶段应用了"conditioning dropout"。也许有一个很好的原因这样做，但我更倾向于认为这只是因为他们决定将这个思想应用于他们已经训练过的模型！）

显然，引导代表了一种权衡：它显著提高了对条件信号的遵循以及整体样本质量，但对多样性的成本很大。然而，在条件生成建模中，这通常是可以接受的权衡：条件信号通常已经捕捉到了我们实际关心的大部分可变性，如果我们希望多样性，也可以简单地修改我们提供的条件信号。
